Private Information Retrieval

Linear homomorphism turns out to be a useful property for building cryptosystems that can do operations on encrypted data

Today, we will be looking at querying remote databases without revealing what we are searching for

  • Suppose a server holds an NNN bit database D∈Z2ND \in \integers_2^ND∈Z2N​
  • A user has an index iii and wants to discover DiD_iDi​ without revealing iii to the server
Private Information Retrieval Scheme

A PIR scheme is a triple of randomized algorithms (Query, Answer, Reconstruct) that meet the following properties, where NNN is the length of the database:

  • Correctness: for any index iii, database DDD, and security parameter nnn, the probability of Reconstruct(Answer(D, qu), st) is at least 1−negl(n)1 - \t{negl}(n)1−negl(n) where (qu, st) are given by Query(n, i)
  • Security: for any two indices i,ji, ji,j, the queries corresponding to iii and jjj are are computationally indistinguishable
  • Succintness: The total bit length of the client's query and the server's answer is less than NNN
    • That is, the total number of bits communicated is smaller than NNN
    • This is to rule out trivial schemes where the user downloads the whole database

Square-Root PIR

Key idea is that the server will represent its NNN bit database as a matrix D∈Z2N×ND \in \integers_2^{\sqrt{N} \times \sqrt{N}}D∈Z2N​×N​​

  • The user wants to read bits at 2D locations (i,j)(i, j)(i,j)
  • Assume we are given a linearly-homomorphic encryption scheme (Gen, Enc, Dec)

Protocol:

  • Query(n,(i,j))\t{Query}(n, (i, j))Query(n,(i,j)):
    • Build the unit vector uju_juj​ that consists of all 0s except for a single 1 at position jjj
    • Sample a secret sk\t{sk}sk with Gen(n)\t{Gen}(n)Gen(n)
    • Output the query vector qu←Enc(sk,uj)\t{qu} \leftarrow \t{Enc}(\t{sk}, u_j)qu←Enc(sk,uj​) along with client-held state st←(sk,i)\t{st} \leftarrow (\t{sk}, i)st←(sk,i)
  • Answer(D,qu)\t{Answer}(D, \t{qu})Answer(D,qu):
    • Output the answer vector ans←D⋅qu\t{ans} \leftarrow D \cdot \t{qu}ans←D⋅qu
  • Reconstruct(ans,st)\t{Reconstruct}(\t{ans}, \t{st})Reconstruct(ans,st):
    • Decrypt the answer vector as v←Dec(sk,ans)v \leftarrow \t{Dec}(\t{sk}, \t{ans})v←Dec(sk,ans)

Note that Query will run on the client, not the server

  • The server is only in charge of addressing Answer

Correctness:

  • If the encryption scheme is linearly homomorphic, we have that ans\t{ans}ans is:
    • D⋅Query(n,(i,j))=D⋅enc(sk,uj)=Enc(sk,D⋅uj)D \cdot \t{Query}(n, (i, j)) = D \cdot \t{enc}(\t{sk}, u_j) = \t{Enc}(\t{sk}, D \cdot u_j)D⋅Query(n,(i,j))=D⋅enc(sk,uj​)=Enc(sk,D⋅uj​)
    • Decoding and getting the iii th bit will then get DijD_{ij}Dij​

Security:

  • Follows from the CPA-security of the encryption scheme

Succintness:

  • Query consists of N\sqrt{N}N​ ciphertexts, as does the output of Answer

Naive Implementation

We attempt to implement with the secret key encryption scheme from LWE

  • Query:
    • We build the unit vector uju_juj​ as described above
    • We sample:
      • A random matrix A∈ZqN×nA \in \integers_q^{\sqrt{N} \times n}A∈ZqN​×n​
      • A secret key vector s∈Zqns \in \integers_q^ns∈Zqn​

      • Query: (A,A⋅a+e+⌊q/2⌋⋅uj)(A, A \cdot a + e + \floor{q/2} \cdot u_j)(A,A⋅a+e+⌊q/2⌋⋅uj​)

    • Output the answer (D⋅A,D⋅b)(D \cdot A, D \cdot b)(D⋅A,D⋅b)
  • Reconstruct:
    • Parse the client-held state as (s,i)(s, i)(s,i)

Optimizations

  • We can use the same AAA matrix as long as we use a random sss and eee each time
    • We can even use AAA as a public parameter known to all users
    • The AAA matrix now no longer needs to be sent around
  • The server can precompute D⋅AD \cdot AD⋅A
  • The user can prefetch the value of D⋅AD \cdot AD⋅A

These make the communication size independent of nnn

  • The PIR scheme now uses log⁡q⋅N\log q \cdot \sqrt{N}logq⋅N​ bits, which is around 16N16 \sqrt{N}16N​ bits
