Encryption Scheme

A symmetric encryption scheme is associated with a key space {Kλ}λ∈N\curly{\mathcal{K}_\lambda}_{\lambda \in \naturals}{Kλ​}λ∈N​, a message space {Mλ}λ∈N\{\mathcal{M}_\lambda\}_{\lambda \in \naturals}{Mλ​}λ∈N​, and a ciphertext space {Cλ}λ∈N\{\mathcal{C}_\lambda\}_{\lambda \in \naturals}{Cλ​}λ∈N​ with two algorithms Enc and Dec with:
Encλ:Kλ×Mλ→Cλ\text{Enc}_\lambda : \mathcal{K}_\lambda \times \mathcal{M}_\lambda \rightarrow \mathcal{C}_\lambdaEncλ​:Kλ​×Mλ​→Cλ​
Decλ:Kλ×Cλ→Mλ\text{Dec}_\lambda : \mathcal{K}_\lambda \times \mathcal{C}_\lambda \rightarrow \mathcal{M}_\lambdaDecλ​:Kλ​×Cλ​→Mλ​

We define the correctness of the encryption scheme if:

Decλ(k,Encλ(k,m))=m\text{Dec}_\lambda(k, \text{Enc}_\lambda(k, m)) = mDecλ​(k,Encλ​(k,m))=m

If our encoders and decoders are probabilistic, then this becomes:

Pr[Decλ(k,Encλ(k,m))=m]=1\prob{\text{Dec}_\lambda(k, \text{Enc}_\lambda(k, m)) = m} = 1Pr[Decλ​(k,Encλ​(k,m))=m]=1

Defining Security

Take 1

An encryption scheme is secure if for every λ\lambdaλ and pair of messages m0,m1m_0, m_1m0​,m1​, we have:
Enc(k,m0)≡Enc(k,m1)\text{Enc}(k, m_0) \equiv \text{Enc}(k, m_1)Enc(k,m0​)≡Enc(k,m1​)
for a random kkk

  • This notation means that the probability distributions, random across the different choices of kkk, are equal for the two messages

A one-time pad that just takes Enc(k,m)=k⊕m\text{Enc}(k, m) = k \oplus mEnc(k,m)=k⊕m, satisfies this

  • However, if the adversary sees more than one message, they can just xor them and get what the xor of the two messages is
    • If they see any plaintext, they can get the key directly out!
  • Here, ciphertexts are functions of the secrete key and can leak information about it
  • We want to be secure even if the adversary sees many ciphertexts
Take 2

An encryption scheme is secure if for every λ,ℓ∈N\lambda, \ell \in \naturalsλ,ℓ∈N and every message m1,…,mℓ,m1′,…,mℓm_1, \dots, m_\ell, m_1', \dots, m_\ellm1​,…,mℓ​,m1′​,…,mℓ​ we have:
(Enc(k,m1),…,Enc(k,mℓ))≡(Enc(k,m1′),…,Enc(k,mℓ′))(\text{Enc}(k, m_1), \dots, \text{Enc}(k, m_\ell)) \equiv (\text{Enc}(k, m_1'), \dots, \text{Enc}(k, m_\ell'))(Enc(k,m1​),…,Enc(k,mℓ​))≡(Enc(k,m1′​),…,Enc(k,mℓ′​))
for a random kkk

This is sufficiently strong, but is impossible to achieve!

  • This cannot be possible if the encryption algorithm is deterministic, since otherwise the adversary can tell if the same message was encrypted twice
  • This is still impossible because the ciphertext contains some information about the secret key kkk, and eventually with enough ciphertexts, all of the information about kkk will be given away

We relax the security requirement, and instead of requiring the distributions to be the same, we require that they only look the same to polynomial time adversaries

  • We refer to this as computationally indistinguishable and use ≈\approx≈ to refer to this
Negligible

A function μ:N→N\mu : \naturals \rightarrow \naturalsμ:N→N is negligible if for every c∈Nc \in \naturalsc∈N, there exists nc∈Nn_c \in \naturalsnc​∈N such that for every n>ncn > n_cn>nc​, we have μ(n)<n−c\mu(n) < n^{-c}μ(n)<n−c

Computationally Indistinguishable

Distributions A\mathcal{A}A and B\mathcal{B}B are computationally indistinguishable if for every probabilistic polynomial time distinguisher D\mathcal{D}D there exists a negliglble function μ\muμ such that:
∣Pr[D(α)=1]−Pr[D(β)=1]∣≤μ(λ)\abs{\prob{\mathcal{D}(\alpha) = 1} - \prob{\mathcal{D}(\beta) = 1}} \leq \mu(\lambda)∣Pr[D(α)=1]−Pr[D(β)=1]∣≤μ(λ)
for random α,β\alpha, \betaα,β drawn from A,B\mathcal{A}, \mathcal{B}A,B

The actual definition is more complicated and even allows the adversary to choose messages adaptively based on previous ciphertexts

CPA Secure

An encryption scheme is secure against adaptively chosen plaintext attacks (CPA secure) if for every PPT adversary A\mathcal{A}A there exists a negligible function μ\muμ such that for every λ\lambdaλ, A\mathcal{A}A wins the following game with probability at most 1/2+μ(λ)1/2 + \mu(\lambda)1/2+μ(λ):

  • The challenger chooses a key k←Kλk \leftarrow \mathcal{K}_\lambdak←Kλ​
  • The adversary, given the length λ\lambdaλ, chooses a message mim_imi​ and receives cic_ici​
    • This can be repeated polynomially many times
  • The adversary A\mathcal{A}A chooses m0,m1m_0, m_1m0​,m1​
  • The challenger chooses a random message out of the two and sends the corresponding ciphertext back
  • The adversary attempts to guess which one of its messages was encrypted

The probability advantage is known as the CPAAdv[A,ε]\text{CPAAdv}[\mc{A}, \varepsilon]CPAAdv[A,ε]

  • This is ∣Pr[W0]−Pr[W1]∣\abs{\prob{W_0} - \prob{W_1}}∣Pr[W0​]−Pr[W1​]∣ where WiW_iWi​ is the event the adversary outputs 111 when the message chosen was iii
  • In the end, we say this is secure if the adversary cannot meaningful gain an advantage in discerning which message was encrypted or not
  • The actual definitions of secure are much stronger
    • CPA-secure is useful as a building block but not one you would ever actually use

Constructing a CPA-Secure Scheme from PRF (Counter-Mode)

If the secret key was infinitely long, we could encrypt all of our messages using a fresh part of the pad, and just tell the user which part of the pad we used

  • As long as we do not reuse the same part of the pad, we will have the same security as the one-time pad
  • To approximate this infinitely long pad, we could use a perfectly random function FFF and pass in a random input to it each time
    • F:{0,1}λ→{0,1}F: \{0, 1\}^\lambda \rightarrow \{0, 1\}F:{0,1}λ→{0,1}
    • We could just generate a random number to pass into FFF every time we want to encrypt a message, and as long as we encrypt significantly less than 2λ/22^{\lambda / 2}2λ/2 messages, we do not expect a collision
    • To this end, we use pseudorandomness
Pseudorandom Functions

A pseudorandom function F:K×X→YF: \mathcal{K} \times \mathcal{X} \rightarrow \mathcal{Y}F:K×X→Y is a function such that for every PPT algorithm A\mathcal{A}A, there exists a negligible function μ\muμ such that:
PRFAdv[A,F]=∣Pr[AF(k,⋅)=1]−Pr[AR=1]∣≤μ\text{PRFAdv}[\mc{A}, \mc{F}] = \abs{\prob{\mathcal{A}^{F(k, \cdot)} = 1} - \prob{\mathcal{A}^R = 1}} \leq \muPRFAdv[A,F]=​Pr[AF(k,⋅)=1]−Pr[AR=1]​≤μ
where kkk is chosen randomly and RRR is a truly random function. The adversary can access FFF and RRR a polynomial number of times and attempt to predict whether or not it is interacting with the pseudorandom function

  • In other words, the adversary cannot in reasonable time distinguish this function from a truly random one

By just using a pseudorandom function in place of the truly random function we get a CPA secure scheme

  • If P=NPP = NPP=NP, then these do not exist
  • This is because you can use circuit-SAT to find out whether or not there exists a key kkk plugged into the F\mc{F}F that results in all of the values you've seen

Formally, this encryption algorithm is known as counter mode, or CTR-mode

CTR-mode Encryption

Given a PRF F\mc{F}F, we can encrypt a series of messages (m1,m2,…,mℓ)(m_1, m_2, \dots, m_\ell)(m1​,m2​,…,mℓ​) where each mim_imi​ is nnn bits long using a key kkk by computing a r←{0,1}nr \leftarrow \curly{0, 1}^nr←{0,1}n and then $outputting:
(r,F(k,r)⊕m1,F(k,r+1)⊕m2,… )(r, F(k, r) \oplus m_1, F(k, r + 1) \oplus m_2, \dots)(r,F(k,r)⊕m1​,F(k,r+1)⊕m2​,…)

For all polynomial adversaries A\mathcal{A}A that make TTT encryption queries of length ℓ\ellℓ messages with blocks of size nnn, there is an adversary B\mathcal{B}B such that:
CPAAdv[A,ε]≤2⋅PRFAdv[B,F]+2T2ℓ2n\text{CPAAdv}[\mc{A}, \varepsilon] \leq 2 \cdot \text{PRFAdv}[\mc{B}, \mc{F}] + \frac{2T^2\ell}{2^n}CPAAdv[A,ε]≤2⋅PRFAdv[B,F]+2n2T2ℓ​

  • If the PRF is secure, then we just need the last term to be negligible, which occurs if T2ℓ<<2nT^2\ell << 2^nT2ℓ<<2n
  • With parallel machines, this can lead to fast decryption
  • If two machines are in communication with each other, they can avoid sending rrr each time by just keeping track of a counter that they each increment

Pseudorandom Permutations (PRPs)

Pseudorandom permutations (PRPs) are an alternative primitive from PRFs

  • Also known as block ciphers
  • The most commonly used block cipher is AES, which replaced DES
    • Both of these are extremely widespread
PRPs

A PRP is defined over a keyspace K\mc{K}K and input space X\mc{X}X and consists of two algorithms PPP and P−1P^{-1}P−1 that map K×X→X\mc{K} \times \mc{X} \rightarrow \mc{X}K×X→X such that:

  • For all keys k∈Kk \in \mc{K}k∈K, P(k,⋅)P(k, \cdot)P(k,⋅) map distinct inputs to distinct outputs

  • P−1(k,⋅)P^{-1}(k, \cdot)P−1(k,⋅) inverts P(k,⋅)P(k, \cdot)P(k,⋅)

  • The PRPAdv[A,P]\t{PRPAdv}[\mc{A}, P]PRPAdv[A,P] is defined as the advantage an adversary can get in a game where it tries to distinguish the PRP from an actual random permutation

    • We say WiW_iWi​ is the probability the adversary says it is a random permutation
  • The bitlength of the input space nnn is the block size

There are many ways to use a PRP to make an encryption scheme, which we will go over

Switching Lemma

We can use a PRP as a PRF if the block size is large enough

  • Then we can just use counter-mode encryption
  • This is the main approach behind AES-GCM
PRP Switching Lemma

Let PPP be a PRP working on inputs of length nnn. For every adversary A\mc{A}A making at most TTT queries, we have:
PRFAdv[A,P]≤PRPAdv[A,P]+T22n\t{PRFAdv}[\mc{A}, P] \leq \t{PRPAdv}[\mc{A}, P] + \frac{T^2}{2^{n}}PRFAdv[A,P]≤PRPAdv[A,P]+2nT2​

If there is an adversary that can break PPP as a PRF with T2<<2nT^2 << 2^nT2<<2n, it can also break it as a PRP with roughly the same advantage

  • If there is no way to break PPP as a PRP, then this therefore means there cannot be anything breaking it as a PRF

This guarantees nothing when T2≈2nT^2 \approx 2^nT2≈2n

  • When this is the case, the adversary can efficiently distinguish a PRP from a PRF
  • This is because at this point, the Birthday Paradox indicates the adversary is likely to see a collision, telling it that it is a PRF and not a PRP
  • Many applications use AES with n=128n = 128n=128 bits, so it is not safe to use the same key more than 2642^64264 times

Cipher-Block Chaining

This is historically important, but should never be used in new systems

  • It is possible to show this is CPA-secure if the underlying pseudo-random permutation PPP is secure

We take a randomly generated c[0]c[0]c[0] (a nonce) and then xor it with m[0]m[0]m[0]

  • We then pass it through the permutation to get out the first ciphertext as well as pass it to xor with m[1]m[1]m[1] and repeat
  • We output all of our ciphertext along with c[0]c[0]c[0] and use the inverse permutation to get the message back

CPAAdv[A,ε]≤2⋅PRPAdv[B,P]+2T2ℓ22n\t{CPAAdv}[A, \varepsilon] \leq 2 \cdot \t{PRPAdv}[\mc{B}, P] + \frac{2T^2\ell^2}{2^n}CPAAdv[A,ε]≤2⋅PRPAdv[B,P]+2n2T2ℓ2​

Disadvantages:

  • Not parallelizable
  • Requires evaluating PRP in forward and inverse directions, which requires more code / hardware
  • Ciphertexts must be a multiple of the block size, leading to padding

For small blocks (i.e. $n = 64$), after seeing enough encrypted messages (i.,e. $2^32$), then it is likely to see two ciphertext blocks that are equal

Even-Mansour Cipher

2n2n2n bit key and nnn bit block size with a public invertible permutation Π\PiΠ

  • The cipheris just PEM((k0,k1),x)=k1⊕Π(x⊕k0)P_{EM}((k_0, k_1), x) = k_1 \oplus \Pi(x \oplus k_0)PEM​((k0​,k1​),x)=k1​⊕Π(x⊕k0​)

If we model Π\PiΠ as truly random, then this has a PRP advantage of 2T2/2n2T^2 / 2^n2T2/2n

AES

AES is an iterated Even-Mansour cipher operating on a key of size 128,192128, 192128,192, or 256256256

  • Block size is fixed at n=128n = 128n=128
  • Derives "round keys" k0,…,krk_0, \dots, k_rk0​,…,kr​ each of length nnn from the input key kkk as a linear function of kkk
  • AES then does:
    • st←x⊕k0st \leftarrow x \oplus k_0st←x⊕k0​
    • For i=1,…,r:i = 1, \dots, r:i=1,…,r:, we do st←Π(x)⊕kist \leftarrow \Pi(x) \oplus k_ist←Π(x)⊕ki​
    • We output ststst
  • For 128128128 bit keys, the number of rounds is set at r=10r = 10r=10

DES

  • In 1985, the Data Encryption Standard was published but never approved for use since its 56 key length was too short
  • AES built on top of a public random permutation
  • DES builds on a pseudorandom function

The main idea is to turn a PRF into a PRP with a Feistel network

  • Uses a 565656 bit keyspace and a 646464 bit block size

  • One round of the Feistel network with PRF FFF is a keyed permutation:

    • πF(k,(x,y))=(y,x⊕F(k,y))\pi_F(k, (x, y)) = (y, x \oplus F(k, y))πF​(k,(x,y))=(y,x⊕F(k,y))
    • This means that we split the input into a left half ($x$) and a right half ($y$)

  • The ℓ\ellℓ round Feistel network applies πF\pi_FπF​ many times:

    • Pℓ((k1,…,kℓ),(x,y))=πF(kℓ,πF(k2,πF(k1,(x,y))))P_\ell((k_1, \dots, k_\ell), (x, y)) = \pi_F(k_\ell, \pi_F(k_2, \pi_F(k_1, (x, y))))Pℓ​((k1​,…,kℓ​),(x,y))=πF​(kℓ​,πF​(k2​,πF​(k1​,(x,y))))

  • It was proven that with ℓ=3\ell = 3ℓ=3, this is sufficient to construct a secure PRP

    • \t{PRPAdv[\mc{A}, P_3]} \leq 3 \cdot \t{PRFAdv}[\mc{B}, F] + \frac{Q^2}{N} + \frac{Q^2}{2N^2}

The DES cipher is just a 16-round Feistel network with a fixed permutation before and after

  • To create the 16 PRFs to use with the Feistel network:
    • 16 round keys k1,…,k16k_1, \dots, k_16k1​,…,k1​6 are all generated as 484848 bit subsets of the bits from the key kkk
    • The round function F(ki,⋅)F(k_i, \cdot)F(ki​,⋅) computes initial state as linear fucntion of round key and input, splits state into six-bit chunks and applies differet non-linear functions (known as S-boxes) to each with a lookup table, and then permutes the bits of the state
    • Picking the S-boxes at random actually makes this cipher weak
      • They were specifically constructed to have nice statistical properties
