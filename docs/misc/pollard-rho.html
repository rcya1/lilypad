Pollard's Rho algorithm is for factoring numbers in time: O(p)O(\sqrt{p})O(p​) where ppp is the smallest prime factor of the number being factored

Let's say we are trying to factor a number N=pqN = pqN=pq

  • If we try to guess the factors, we have a 2/(N−1)2 / (N - 1)2/(N−1) chance each time to find the factor

Applying the Birthday Paradox:

  • Instead of picking just one number, we pick kkk numbers
  • We ask if xi−xjx_i - x_jxi​−xj​ divides NNN for some i,ji, ji,j
  • For k≈nk \approx \sqrt{n}k≈n​, we have around a 1/21/21/2 chance to pick a factor
    • However, this by itself doesn't save any effort since we need to do k2k^2k2 pairwise comparisons / divisions
  • Instead we look to see if there if \tgcd(xi−xj,N)>1\t{gcd}(x_i - x_j, N) > 1\tgcd(xi​−xj​,N)>1
    • Instead of looking for just ppp and qqq, we look for p,2p,3p,…,(q−1)p,q,…p, 2p, 3p, \dots, (q-1)p, q, \dotsp,2p,3p,…,(q−1)p,q,…
    • We have p+q−2p + q - 2p+q−2 numbers we can look for
  • Now we need a kkk that is on the order of N1/4N^{1/4}N1/4 and do pairwise comparisons
    • But if NNN is very large, we would have to store this many numbers in memory

We use a pseudo random number generator f(x)=x2+a(mod  n)f(x) = x^2 + a \pmod{n}f(x)=x2+a(modn) for a randomly chosen aaa

  • We successively apply xn+1=f(xn)x_{n+1} = f(x_n)xn+1​=f(xn​)
  • We then check consecutive numbers and see if their difference shares factors with NNN
  • However, we will eventually cycle
    • We could store all numbers in memory, but this goes back to the original problem
    • Instead, we use Floyd's cycle detection algorithm
  • This has a good probability of finding a factor if it exists, otherwise after a few tries we can give up
